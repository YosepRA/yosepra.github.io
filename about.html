Skills

HTML

The structure of the web. Semantic structure is in my rule book. I can use HTML5 semantic tags to create meaningful and scalable website.

CSS

The styling of the web. I am proficient with how browsers lay out its elements. I understand the behavior of CSS box system and to control its flow utilizing the latest technologies from CSS3 such as Flexbox and Grid system. With my understanding on using these technologies, the dependencies on external libraries will be decreased significantly. I can also use SASS to excel even further to create structured and maintainable stylesheets.

JavaScript

The programming language of the web. The first language that I’m good at. It defines how users interact with the website. I’m adept to the latest ECMAScript standard that is necessary for modern apps. I love creating websites that feel more like an app than a static page. Using technologies such as AJAX to create a seamless interaction with little dependencies on page reloads is fun to me. Well, solving problem is always fun for me anyway so programming will provide me a platform to play around.

Bootstrap

My go-to front-end framework. I can use its components to either quickly create prototypes, website templates, or even a full website. Although I will choose vanilla CSS and JavaScript over Bootstrap whenever I can, I still can work with this framework with ease.

jQuery

Cross-browser adaptability has always been a concern since forever, and jQuery is the soundest answer for that. But with the decreasing concern of cross-browser with browsers are getting more and more universal, I settle with vanilla JavaScript most of the time.

Node js

Node is my back-end environment. For now, I only have a basic ability to utilize this marvelous technology to create website with features such as authentication. Having a back-end will surely help a website/app to rise to even greater height. Despite my current focus on front-end development, I often use node to create scripts for automating my workflow. It surely is fun to play with it and having to master it sometime in the future is nothing but pleasure to me.

Express js

A minimalist web application framework. It takes off the basics (and tedious) job to set up a back-end environment for me to get started building things.

MongoDB

The database of my choosing. I’ve tried SQL long ago and few differences between these two databases put me in favor of MongoDB. MongoDB has an easier approach to me since it’s basically storing some sort of JSON file in it. The process of saving and retrieving is also as straightforward as ever. I will surely dig deeper on the technology along with other back-end technologies that I intend to master.

Adobe Photoshop

There are only so much you can do with CSS. Creating custom graphic assets for a website using Photoshop will provide more options in both layout and imagery.

Adobe Illustrator

The sibling of Adobe Photoshop. There are assets such as logo that will need to have the highest quality all across different screen sizes. SVG will provide a solution just for that. Utilizing Illustrator to create vector graphics will make a website look more professional. I mean just imagine your logo got pixelated in huge screen? It’s not cool anymore.
